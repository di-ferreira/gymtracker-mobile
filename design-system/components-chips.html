Chips

Chips são elementos compactos que representam filtros, ações rápidas, informações ou seleções removíveis. Quatro variantes cobrem todos os contextos do GymTracker — de filtros de grupo muscular a tags de séries e cargas.

Filtro Chip

Chips de alternância usados para filtrar treinos por grupo muscular. Tocar alterna entre os estados padrão e selecionado. mobile web

Interativo — clique para alternar

Estados

Padrão (desligado)
Selecionado
Hover
Selecionado + Hover

Ação Chip

Chips com ícone e rótulo para ações rápidas. Usados em barras de ação e menus de contexto. mobile web

Estados

Padrão
Hover
Pressionado

Informação Chip

Chips não interativos que exibem valores como séries, repetições e cargas. Usados para exibir estatísticas e tags de exercício. ambos

Nota: Info Chips são puramente exibitivos. Não respondem a cliques, hover ou foco. Use no HTML — nunca

Removível Chip

Chips com botão de fechamento (X) usados em seleções de filtro ativas. O botão de exclusão permite remover filtros selecionados. mobile web

API

Propriedades e eventos disponíveis para cada variante de Chip.

Prop Tipo Padrão Descrição
variant filter | action | info | deletable filter Define o tipo visual e comportamental do chip
selected boolean false Estado selecionado do Filtro Chip (toggle)
icon ReactNode | string — Ícone exibido à esquerda do rótulo (Ação Chip)
label string — Texto do chip (obrigatório)
onDelete (() => void) — Callback ao clicar no botão X (Removível Chip)
disabled boolean false Desabilita interação (Filter, Action, Deletable)
onPress (() => void) — Callback ao pressionar o chip (Ação Chip)
onToggle ((sel: boolean) => void) — Callback ao alternar seleção (Filtro Chip)
accessibilityLabel string label Rótulo de acessibilidade para leitores de tela

Regras de Uso

Diretrizes para manter consistência e acessibilidade.

Regra Descrição
Filtro Chip para múltipla seleção Múltiplos Filtro Chips podem ser selecionados simultaneamente (ex: filtrar por Peito E Costas). Nunca use Filtro Chip para seleção única — use Tabs ou Seletores.
Ação Chip para ações simples Ação Chips devem disparar ações imediatas. Nunca use Ação Chip como toggle — para isso, use Filtro Chip.
Info Chip não é interativo Info Chips nunca devem ter hover, foco ou cursor pointer. Use no HTML, nunca
Removível Chip com acessibilidade O botão X deve ter aria-label descritivo (ex: "Remover filtro Peito"). O chip container não é focusable — apenas o botão X recebe foco.
Alvo de toque mobile No mobile, todos Chips interativos devem ter área de toque mínima de 44×44px. Use padding adicional para atingir o mínimo sem alterar a altura visual de 32px.
Texto conciso Máximo de 20 caracteres no rótulo. Para textos longos, use truncamento com reticências. Rótulos de grupo muscular devem ser palavras únicas.
Espaçamento entre Chips Sempre use gap de 8px (var(--gt-space-2)) entre chips. Nunca use margem individual — use flexbox com gap no container.
Não misture variantes na mesma linha Evite misturar Filtro Chips e Ação Chips na mesma fileira. Cada variante tem propósito distinto e deve ser agrupada separadamente.

Fazer / Não Fazer

Erros comuns e melhores práticas ao utilizar Chips.

Fazer

Use Filtro Chips para permitir que o usuário selecione múltiplos grupos musculares simultaneamente.

Peito Costas Pernas

Não Fazer

Não use Chips para seleção exclusiva. Para escolha única, use Seletores ou Tabs segmentados.

Sim Não

Fazer

Use Info Chips para exibir valores de séries e cargas de forma compacta ao lado do nome do exercício.

3x12 80kg

Não Fazer

Não coloque Info Chips como botões clicáveis. Eles são puramente exibitivos.

3x12

Fazer

No mobile, garanta alvo de toque mínimo de 44×44px usando padding adicional invisível.

Peito

Não Fazer

Não use Chips com área de toque menor que 44px no mobile — dificulta a interação por toque.

Exemplo de Código

Implementação dos Chips com React Native e CSS.

/* ── Chips ── */

.gt-chip {
  display: inline-flex;
  align-items: center;
  justify-content: center;
  height: 32px;
  padding: 0 var(--gt-space-3);
  border-radius: var(--gt-radius-full);
  font-family: var(--gt-font-body);
  font-size: 13px;
  font-weight: 500;
  border: 1px solid var(--gt-border);
  background: var(--gt-surface-2);
  color: var(--gt-fg-secondary);
  cursor: pointer;
  transition: all 100ms cubic-bezier(0.25, 0.1, 0.25, 1);
  user-select: none;
  white-space: nowrap;
  outline: none;
}

.gt-chip:hover {
  border-color: var(--gt-border-strong);
}

.gt-chip-selected {
  background: var(--gt-primary);
  color: var(--gt-bg);
  border-color: var(--gt-primary);
}

.gt-chip-selected:hover {
  background: var(--gt-primary-hover);
  border-color: var(--gt-primary-hover);
}

.gt-chip:focus-visible {
  outline: 2px solid var(--gt-primary);
  outline-offset: 2px;
}

/* Ação Chip */
.gt-chip-action {
  gap: var(--gt-space-2);
}
.gt-chip-action:active {
  transform: scale(0.97);
}

/* Info Chip (não interativo) */
.gt-chip-info {
  cursor: default;
  background: var(--gt-surface-2);
  border: 1px solid var(--gt-border);
  color: var(--gt-fg-secondary);
}
.gt-chip-info:hover {
  border-color: var(--gt-border);
}

/* Removível Chip */
.gt-chip-deletable {
  padding-right: 8px;
  gap: var(--gt-space-1);
}
.gt-chip-delete {
  display: inline-flex;
  align-items: center;
  justify-content: center;
  width: 16px;
  height: 16px;
  border-radius: 50%;
  background: transparent;
  border: none;
  color: var(--gt-fg-tertiary);
  cursor: pointer;
  transition: all 100ms cubic-bezier(0.25, 0.1, 0.25, 1);
  flex-shrink: 0;
  padding: 0;
  font-size: 10px;
  margin-left: 4px;
}
.gt-chip-delete:hover {
  background: rgba(255, 255, 255, 0.15);
  color: var(--gt-fg);
}
.gt-chip-delete:focus-visible {
  outline: 2px solid var(--gt-primary);
  outline-offset: 1px;
}

/* Container de Chips */
.chip-group {
  display: flex;
  flex-wrap: wrap;
  gap: var(--gt-space-2);
}

/* Mobile: alvo de toque mínimo */
@media (pointer: coarse) {
  .gt-chip:not(.gt-chip-info) {
    min-height: 44px;
    padding: 6px var(--gt-space-3);
  }
}
