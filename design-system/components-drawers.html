Drawers

Drawers são painéis laterais ou inferiores que emergem da borda da tela. Usados para navegação, filtros e ações contextuais sem abandonar a tela atual.

Variantes

Três tipos de drawer cobrem os contextos de navegação, filtragem e uso mobile. Cada um com largura, posição e comportamento específicos.

Navigation Drawer
Painel esquerdo para navegação do dashboard
Web
Filter Drawer
Painel direito para filtros de tabela/lista
Web
Mobile Drawer
Bottom sheet para ações contextuais
Mobile

Demonstração Interativa

Clique nos botões abaixo para abrir cada tipo de drawer. Feche clicando no overlay, no botão X ou (no mobile) deslizando para baixo.

API

Propriedades do componente Drawer para implementação em React Native e Next.js.

Prop Tipo Padrão Descrição
type 'nav' | 'filter' | 'mobile' 'nav' Tipo do drawer
open boolean false Controla se o drawer está aberto
onClose () => void — Callback ao fechar o drawer
title string — Título exibido no header (20px semibold)
overlay boolean true Exibe overlay escuro ao fundo
swipeToClose boolean false Permite fechar deslizando para baixo (mobile)
width number 280 / 320 / 100% Largura customizada do drawer
maxHeight number | string '70vh' (mobile) Altura máxima (apenas mobile)
children ReactNode — Conteúdo interno do drawer

Regras de Uso

Princípios para manter consistência na utilização de drawers.

Regra Descrição
Sempre com overlay Drawers devem ter overlay escuro rgba(0,0,0,0.5). O clique no overlay fecha o drawer.
Um drawer por vez Nunca empilhe drawers. Feche o atual antes de abrir outro.
Animação consistente Sempre use slide-in de 300ms com --gt-ease-decelerate. Nunca use animações instantâneas ou muito lentas.
Header com fechamento Todo drawer deve ter header com título (20px semibold) e botão de fechar (X). Nunca remova o botão de fechar.
Mobile: swipe down No mobile drawer, permita fechar deslizando para baixo. Use um handle visual (barra) no topo como indicador.
Foco na acessibilidade Ao abrir, mova o foco para o drawer. Ao fechar, retorne o foco para o elemento ativador. Use aria-modal e role="dialog".
Navigation drawer: itens iguais à navegação Os itens do navigation drawer devem ser idênticos ao componente de navegação (mesmos ícones, textos e estados ativos).
Filter drawer: botões Aplicar/Limpar O filter drawer deve conter botões "Aplicar" (primário) e "Limpar" (secundário) no footer. Nunca omita o botão de reset.

Tokens CSS

Custom properties para implementar drawers. Copie e use no projeto.

/* ── Drawers ── */

/* Tipos */
--gt-drawer-nav-width: 280px;
--gt-drawer-filter-width: 320px;
--gt-drawer-mobile-width: 100%;
--gt-drawer-mobile-max-height: 70vh;

/* Superfície */
--gt-drawer-bg: var(--gt-surface);
--gt-drawer-border: 1px solid var(--gt-border);
--gt-drawer-overlay: rgba(0, 0, 0, 0.5);

/* Header */
--gt-drawer-header-title: 20px;
--gt-drawer-header-weight: 600;
--gt-drawer-close-size: 32px;
--gt-drawer-close-radius: var(--gt-radius-md);

/* Animação */
--gt-drawer-duration: 300ms;
--gt-drawer-ease: cubic-bezier(0, 0, 0.2, 1);

/* Filter drawer inputs */
--gt-drawer-input-height: 36px;
--gt-drawer-input-bg: var(--gt-surface-2);
--gt-drawer-input-border: var(--gt-border);
--gt-drawer-input-focus: var(--gt-primary);
--gt-drawer-input-radius: var(--gt-radius-md);

/* Footer buttons */
--gt-drawer-footer-btn-height: 36px;
--gt-drawer-footer-gap: var(--gt-space-3);
