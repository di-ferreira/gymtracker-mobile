Botões

Os botões são o principal elemento de interação do GymTracker. Quatro variantes, três tamanhos e estados interativos garantem consistência entre mobile e web.

Variantes

Cada variante comunica um nível de ênfase diferente. Use Primary para a ação principal, Secondary para ações complementares, Ghost para baixa prioridade e Danger para ações destrutivas.

Mobile vs Web

Regras específicas por plataforma para garantir a melhor experiência em cada contexto de uso.

Atenção: Botões devem seguir as diretrizes de acessibilidade de cada plataforma. No mobile, o alvo mínimo de toque é obrigatório; na web, o foco por teclado é essencial.

Mobile (React Native)

Alvo de toque mínimo de 44 × 44px — mesmo para botões sm (32px de altura visual), a área de toque deve ter pelo menos 44px de altura com padding adicional.

Use hitSlop no React Native para expandir a área de toque sem alterar o layout visual.

Web (Next.js)

Foco visível por teclado: anel de 2px solid #FF7A1A com 2px de offset. Sempre use :focus-visible para não mostrar anel em cliques de mouse.

Suporte a Tab e Enter/Space nativamente via elemento

Regras de Uso

Princípios para manter consistência e clareza na utilização de botões.

Regra Descrição
Um Primary por seção No máximo 1 botão Primary por seção ou tela. O Primary deve ser a ação mais importante e evidente.
Ghost para baixa prioridade Use Ghost apenas para ações terciárias (ex: "Saiba mais", "Fechar"). Nunca use Ghost como ação principal.
Danger requer confirmação Ações destrutivas (excluir, remover) devem usar Danger e exigir confirmação em modal antes de executar.
Botões com ícone Ícone à esquerda do texto, nunca à direita. Use o gap do token correspondente ao tamanho. Ícones devem ter 16px (sm), 18px (md) ou 20px (lg).
Botão ícone-only Permitido apenas para ações universalmente reconhecidas (fechar, voltar, buscar). Use o mesmo tamanho do botão e borda circular.aria-label obrigatório.
Nunca misture variantes Não crie botões estilizados fora das 4 variantes. Não use Primary com borda ou Danger sem fundo semântico.
Texto conciso Máximo de 3 palavras no rótulo. Use verbos de ação: "Salvar", "Excluir", "Cancelar". Nunca use texto genérico como "OK" ou "Clique aqui".
Loading bloqueia interação No estado loading, o botão deve desabilitar nova interação (pointer-events: none) e ocultar o texto, exibindo apenas o spinner.
