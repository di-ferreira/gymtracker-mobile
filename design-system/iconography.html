Estilo

Ícones do GymTracker seguem o estilo monoline com traço consistente, caps e joins arredondados, e herança de cor via currentColor.

Grade e Padding

Ícones são desenhados em uma grade de 24px com 2px de padding interno, garantindo que o peso óptico esteja centralizado. A área visível do ícone é 20px dentro da grade de 24px.

Área do ícone (20px)
Padding (2px)

Grade: 24 × 24px

Padding: 2px em cada lado

Área visível: 20 × 20px

O padding garante centralização óptica para ícones com elementos que se estendem até as bordas (ex: setas, circunflexos).

Biblioteca de Ícones Recomendada

Bibliotecas com a mesma linguagem visual (monoline, 1.5–1.8px de stroke) para ambas as plataformas.

Phosphor Icons
React Native

Use react-native-phosphor. Biblioteca flexível com variantes thin, light, regular, bold, fill. Use a variante regular como padrão e fill para estado ativo na navegação.

Lucide
Next.js

Use lucide-react. Ícones monoline com 1.5px de stroke por padrão. Visual consistente com Phosphor — mesma linguagem de design entre mobile e web.

Regras de Uso

Diretrizes para uso consistente de ícones no GymTracker.

  • Nunca use emoji como ícones — substitua por ícones SVG da biblioteca (sem ✨🚀🎯⚡🔥💡)
  • Ícones ao lado de labels devem ter o mesmo tamanho ou um passo menor que o texto
  • Áreas de toque: mínimo de 44px em mobile (adicione padding na área de hit além do ícone)
  • Ícone de navegação ativa: troque para variante filled
  • Alinhamento de stroke consistente dentro de uma mesma tela — não misture ícones thin com bold
