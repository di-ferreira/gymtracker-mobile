Regras de Uso

Diretrizes obrigatórias para manter consistência e acessibilidade em toda a interface.

  • Nunca use floating labels — label sempre visível acima do campo para evitar confusão de acessibilidade
  • Todo input obrigatório deve exibir indicador visual e mensagem de erro clara em pt-BR
  • Placeholder nunca substitui label — placeholder é texto temporário, label é permanente
  • Campos numéricos de treino (peso, reps, carga) devem usar teclado numérico no mobile e inputmode="decimal" no web
  • Inputs desabilitados usam opacity 0.5 — nunca remova o campo do DOM, pois isso quebra a navegação por tela
  • Campos readonly mantêm a estrutura visual mas reduzem a cor do texto para #888888 e fundo para #0A0A0A
  • Largura máxima de 480px para campos isolados — inputs full-width apenas em grids de formulários
  • Agrupe campos relacionados visualmente com labels e texto auxiliar — nunca empilhe inputs sem contexto
  • Estado de erro sempre inclui borda vermelha (#EF4444), box-shadow e mensagem de erro abaixo do campo
  • Transições de estado usam duration 100ms com ease-standard para feedback instantâneo sem sensação de lag
