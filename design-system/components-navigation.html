Navegação

Componentes de navegação do dashboard web: barra superior, navegação lateral e breadcrumbs. Projetados para fornecer orientação espacial e acesso rápido a todas as seções do painel.

Barra Superior (Top Bar)

Cabeçalho fixo do dashboard com logo, breadcrumb, campo de busca e avatar do usuário. Fornece contexto da página atual e acesso global a ações.

Responsividade

Comportamento da navegação em diferentes tamanhos de tela.

Atenção: Em telas menores que 768px, a barra superior simplifica (breadcrumb e search movem-se para dentro do conteúdo) e a navegação lateral torna-se um drawer acessível por botão de menu.

Breakpoint Top Bar Side Nav
≥ 1024px Completa: logo + breadcrumb + search + avatar Expandida por padrão (260px)
768px – 1023px Completa: logo + breadcrumb + search + avatar Recolhida por padrão (72px), expandível ao passar o mouse
< 768px Simplificada: logo + ícone menu + avatar Oculta, abre como drawer com overlay escuro

Regras de Uso

Diretrizes para manter a navegação consistente e acessível.

Regra Descrição
Sidebar persistente A navegação lateral deve estar sempre visível em telas ≥ 768px. Nunca oculte a sidebar sem ação explícita do usuário.
Um item ativo Apenas um item pode estar ativo por vez. O item ativo recebe borda esquerda primária de 2px e fundo tingido com rgba(255,122,26,0.06).
Avatar com iniciais Se não houver imagem de perfil, use as iniciais do nome em círculo de 32px com fundo --gt-surface-3 e borda --gt-border.
Breadcrumb obrigatório Toda página interna do dashboard deve exibir breadcrumb. Páginas raiz (Dashboard) podem exibir apenas o rótulo atual.
Separador de breadcrumb Use chevron (›) como separador, nunca barra (/). O separador usa cor --gt-fg-muted.
Busca global O campo de busca na top bar busca em todo o sistema (exercícios, treinos, histórico). Use 100ms de debounce.
Drawer no mobile Em telas < 768px, a sidebar abre como drawer com overlay escuro (rgba(0,0,0,0.5)) e animação 200ms ease-decelerate pela esquerda.
Navegação por teclado Itens da sidebar devem ser focáveis com Tab. Enter/Space ativam o item. A top bar deve suportar Tab para busca e avatar.
Transição de collapse A transição entre expandido e recolhido usa 200ms ease-standard. Rótulos ocultam com opacity 0 — nunca use display: none.
